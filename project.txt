HIGH-POWER DESIGN IDEA (VERY ADVANCED)
1) domain/ = Pure business rules (no DB/no API)
2) application/ = Use-cases (commands/queries/sagas/workflows) + Ports (interfaces)
3) infrastructure/ = Adapters (DB, messaging, cache, auth, http clients) + outbox pattern
4) interfaces/ = Delivery layer (API/Workers/CLI) only, keeps framework code here
5) contracts/ = Versioned event/command schemas (safe evolution for distributed systems)
6) plugins/ = Dynamic extensions (optional features without touching core)
7) testsupport/ = Fakes/factories to write very fast, reliable tests

ENTRYPOINTS (TYPICAL)
- API: src/ultra_system/interfaces/api/asgi.py
- Worker: src/ultra_system/interfaces/workers/runner.py
- CLI: src/ultra_system/interfaces/cli/main.py

NOTE
- Yeh structure extremely scalable hai: monolith bhi, microservices bhi, dono ke liye.
- Sirf Python packages/structure diya hai, aap apne framework (FastAPI/Django/Starlette) ko interfaces/api me fit kar sakte ho.
